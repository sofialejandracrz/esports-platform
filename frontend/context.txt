- Dashboard para administrar cada parte del sistema.
- Inicio de sesión y registro basado en roles (Usuario y Administrador, solo administradores pueden registrar otros administradores. Todo por JWT. Para las entidades se debe hacer una normalización usando una entidad principal “Persona”, cada usuario tiene un nickname único). Estar registrado no será necesario para visualizar el sistema, en el home deben estar los botones respectivos para cada formulario.
- Página para visualizar el perfil de cada usuario, ubicado en usuario/perfil/”nickname”. Tendrá un resumen del usuario, dinero ganado (saldo), su lista de amigos, vitrina de trofeos, XP (Se consigue en torneos o desafíos), logros y estadísticas completas por juego. El usuario registrado verá esa información y todo lo necesario para administrar su perfil, si otra persona visualiza su perfil verá un botón de agregar amigo.
- Sistema de amigos entre usuarios y seguidores tipo Instagram.
- Dashboard para los usuarios: ubicado en /usuario/configuracion. “configuracion” mostrará la información personal pero el dashboard incluirá un layout con un sidebar para navegar a las siguientes secciones de configuración: /usuario/configuración/personal (La primera vista, nickname no modificable, sobre ti, género, zona horaria), /usuario/configuración/social (enlaces de las redes sociales del usuario), /usuario/configuración/juegos (identificador de las cuentas de juegos del usuario. Playstation Network, Xbox Live, Epic, Activision, Rogue ID, Apex Legends ID, Steam, Riot ID, Ubisoft ID, EA ID), /usuario/configuración/preferencias (Habilitar o deshabilitar desafíos, true/false), /usuario/configuración/cuenta (Ver correo y cambio de contraseña), /usuario/configuración/seguridad (Dirección de paypal, nombre de pila, apellido, Teléfono, dirección, ciudad, estado, código postal, país, divisa,), /usuario/configuración/retiro (Esto será una integración a futuro)
- Registro de juegos (Cuando se construya el dashboard admin), cada juego tiene: Plataforma/s (PlayStation, Xbox, PC, Móvil, Multiplataforma), Modo/s de juego (disponibles dentro del juego).
- Sistema de créditos, comprados con dinero real, se usan para entrar a torneos, también añadir saldo (expresado en dólares $) que simule dinero real para compras dentro del sistema, este saldo es recargable con dinero real. El saldo se puede retirar al cumplir un mínimo de retiro.
- Creación y administración de torneos. Cada torneo tiene: 
•	1. Información básica: Título, descripción, Fechas (Inicio de registro, Fin de registro, Inicio del torneo).
•	2. Detalles del torneo: Juego, Plataforma, Modo de juego (Cada torneo está asociado a un juego ya registrado en el sistema, de ahí se obtiene la plataforma dónde se jugará y el modo de juego), Región (Global, Norteamérica, Europa, etc.), Tipo de torneo (Eliminación simple por default, en futuras actualizaciones se agregarán más tipos), Al mejor de (1, 3, 5, 7), Formato (1 contra 1, etc.), Torneo cerrado (solo por invitación), true o false) Conjunto de reglas (Básicas del juego o personalizadas, personalizadas se guardará como cadena de texto); Opciones adicionales: Jugadores de PC permitidos (true, false), Se requiere transmisión (true, false), Se requiere cámara web (true, false); Tipo de entrada (mando, teclado y ratón, todos), Capacidad de inscripciones (Inscripciones máximas que tendrá el torneo).
•	3. Premios: Cuota de inscripción: Precio por jugador (créditos, opcional si el acceso es gratis), Total del fondo de premios, Fondo de premios después de la comisión del anfitrión, comisión del anfitrión (Porcentaje), comisión total del anfitrión (el porcentaje expresado en su totalidad); Distribución de premios: 1.er lugar y 2.do lugar (porcentaje del fondo de premios). Cada premio se representa en saldo, pero se calcula en base a los créditos que se acumularon con los jugadores registrados.
•	4. Detalles del anfitrión: ¿Cómo pueden ponerse en contacto con el anfitrión? (cadena de caracteres, expresar el campo de una forma más resumida), URL de enlaces a redes sociales (opcional, twitch, discord, youtube, facebook, x), Servidor de Discord (para mejor administración del torneo); El anfitrión es el usuario que publica el torneo.
•	5. Gráficos: Banner personalizado para torneos (imagen, opcional), Miniatura (imagen, opcional).
•	6. Finalizar torneo: Ya se ha recopilado toda la información y se puede publicar. El estado automáticamente pasará a “Próximamente”, los estados son: Próximamente, en curso, terminado.
Además, cada torneo debe tener un tipo de torneo asignado, este tipo de torneo se usará también para definir el tipo de premio/trofeo que ganará el usuario y el cuál se visualizará el conteo de los totales ganados en su perfil.
